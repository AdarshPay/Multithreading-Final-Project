
We have a singularity container which has CUDA, Julia, Python, PyTorch, etc.
It's in a .sif file called taccjulia.sif
Download it from the dropbox in the discord to your local then scp it to your working directory,
or you can wget it to your working directory directly.


====== BRINGING UP THE CONTAINER ======
IDEA: 
Entering the container in a shell. Binding our github repo directory to the container!
	$ apptainer shell --nv --bind (host_path):(container_path) (singularity file)

DO:
1. $ cdw or cd $WORK

2. Download the tgz and unzip it into your working directory and into your repo's directory.

3. Your working directory should be $WORK/Multithreading-Final-Project

4. Unzip the tgz file so you just have taccjulia.sif

5. $ idev -p rtx

6. $ module load tacc-apptainer

7. $ apptainer shell --nv --bind $WORK/Multithreading-Final-Project:/workspace taccjulia.sif

You're now in a shell prompt with all of the dependencies!
=======================================


====== OTHER COMMANDS ======
Instead of doing the apptainer shell command, you can just run the below one to dispatch a job to the container
apptainer exec --nv myjulia.sif (a command like julia, nvidia-smi, python3  -c "import torchl print(torch.cuda.is_available())")



====== ENSURING IT WORKED ======
apptainer exec --nv taccjulia.sif nvidia-smi
apptainer exec --nv taccjulia.sif julia
apptainer exec --nv taccjulia.sif julia run_cuda.jl // replace run_cuda.jl with some julia kernel we use cuda in
apptainer exec --nv taccjulia.sif python3 -c "import torch; print(torch.cuda.is_available())"
